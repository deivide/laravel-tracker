## Project Overview

PragmaRX Tracker is a comprehensive Laravel visitor tracking package that gathers detailed analytics about application requests, users, devices, and site visits. It provides automatic tracking of sessions, page views, user agents, devices, geographic locations, database queries, application events, errors, and more. Includes an optional web-based analytics dashboard.

## Quick Reference

- **Tech Stack:** PHP >=7.0, Laravel 5-11, Eloquent ORM, Doctrine DBAL
- **Entry Point:** `src/Vendor/Laravel/ServiceProvider.php` (bootstrap)
- **Architecture Pattern:** Repository Pattern with 25+ repositories, 29 Eloquent models, Facade access
- **Database:** Dedicated `tracker` connection, 25+ tables (tracker_* prefix)
- **Key Dependencies:** jenssegers/agent, ua-parser/uap-php, jaybizzle/crawler-detect, snowplow/referer-parser, geoip2/geoip2 (optional)

## Getting Started

### Prerequisites

- PHP >= 7.0
- Composer
- A Laravel application (5.x through 11.x)

### Setup

```bash
composer require deivide/tracker
php artisan vendor:publish --provider="PragmaRX\Tracker\Vendor\Laravel\ServiceProvider"
php artisan tracker:tables
php artisan migrate
```

### Run Tests

```bash
./vendor/bin/phpunit
```
